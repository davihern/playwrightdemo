## 🚀 Running Your Own Analysis

See `PMD_CPD_USAGE_GUIDE.md` for complete instructions.

**Quick start:**

```bash
# Download PMD (if not already installed)
cd /tmp
wget https://github.com/pmd/pmd/releases/download/pmd_releases%2F7.9.0/pmd-dist-7.9.0-bin.zip
unzip pmd-dist-7.9.0-bin.zip

# Run analysis
/tmp/pmd-bin-7.9.0/bin/pmd cpd \
  --minimum-tokens 50 \
  --language typescript \
  --dir . \
  --no-fail-on-error \
  --format text
```

## 🤔 Understanding Duplications

**Not all duplications are bad!** Some are acceptable:
- ✅ Small boilerplate code (< 30 tokens)
- ✅ Configuration patterns
- ✅ Test assertion patterns in comprehensive tests

**Focus on refactoring:**
- 🔴 Complete file duplications
- 🔴 Large code blocks (100+ tokens)
- 🔴 Business logic duplications
- 🟡 Test setup code (can use fixtures)
- 🟡 Medium code blocks (50-100 tokens)
